## Description
  
FETCH variable count does not match cursor column count.
  
## Summary
  
The number of variables in a FETCH statement should match the cursor's SELECT column count.
  
### Examples
  
```sql
CREATE PROCEDURE [dbo].[TestFetchMismatch]
AS
BEGIN
    SET NOCOUNT ON;
    DECLARE @id INT, @name NVARCHAR(100), @extra INT;
    DECLARE cur CURSOR FOR
        SELECT [object_id], [name] FROM [sys].[objects];
    OPEN cur;
    FETCH NEXT FROM cur INTO @id, @name, @extra;
    CLOSE cur;
    DEALLOCATE cur;
END;

-- SRD0077
```
  
### Remarks
  
When the number of variables in FETCH INTO does not match the number of columns in the
cursor's SELECT statement, it causes a runtime error. This rule detects mismatches
between cursor declaration column count and FETCH INTO variable count.
