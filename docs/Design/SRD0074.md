## Description
  
Avoid weak hashing algorithms (MD2, MD4, MD5, SHA, SHA1). Use SHA2_256 or SHA2_512.
  
## Summary
  
Weak hashing algorithms (MD2, MD4, MD5, SHA, SHA1) should not be used.
  
### Examples
  
```sql
CREATE PROCEDURE [dbo].[TestWeakHashing]
AS
BEGIN
    SET NOCOUNT ON;
    DECLARE @hash VARBINARY(8000);
    SET @hash = HASHBYTES('MD5', 'test data');

    DECLARE @hash2 VARBINARY(8000);
    SET @hash2 = HASHBYTES('SHA1', 'test data');
END;

-- SRD0074
```
  
### Remarks
  
HASHBYTES with MD2, MD4, MD5, SHA, or SHA1 uses weak hashing algorithms that are
vulnerable to collision attacks. Use SHA2_256 or SHA2_512 instead.
