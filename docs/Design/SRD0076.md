## Description
  
Identical expressions on both sides of a comparison operator.
  
## Summary
  
Identical expressions should not be used on both sides of a binary operator.
  
### Examples
  
```sql
CREATE PROCEDURE [dbo].[TestIdenticalExpressions]
AS
BEGIN
    SET NOCOUNT ON;
    DECLARE @x INT = 5;
    IF @x = @x
    BEGIN
        PRINT 'Always true';
    END;
END;

-- SRD0076
```
  
### Remarks
  
Expressions like WHERE col = col or col &lt;&gt; col are almost always bugs.
If comparing a column to itself is intended (e.g., to find non-NULL values),
use IS NOT NULL instead.
