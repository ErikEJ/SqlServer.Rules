# SQL Server Rule: SRD0095
  
|    |    |

## Rule Information
  
|    |    |
|----|----|
| Id | SRD0095 |

| Category | Design |

| Applicable Types | Procedure  |
|   | Scalar Function |

## Description
  
Check constraints on temp tables should not be named.
  
## Summary
  
Check constraints on temp tables should not be named.
  
### Examples
  
```sql
CREATE PROCEDURE [dbo].[SRD0095TempTableWithNamedCheckConstraint]
AS
BEGIN
    SET NOCOUNT ON;

    CREATE TABLE #TempOk
    (
        ColA INT CHECK (ColA > 1)
    );

    CREATE TABLE #TempBad
    (
        ColA INT CONSTRAINT [CK_TempBad_ColA] CHECK (ColA > 1)
    );
END

-- SRD0095
```
```sql
CREATE PROCEDURE [dbo].[TempTableWithNamedCheckConstraint]
	
AS
	Set nocount on;


	Create table #1
	(
		cola integer check(cola > 1)
	)

	Create table #2
	(
		cola integer Constraint [chk] check (cola > 1)
	);

RETURN 0

-- SRD0095
```
  
### Remarks
  
Naming check constraints on temporary tables can cause contention issues in tempdb.
